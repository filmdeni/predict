# Product

## Register

product

## Users

คนไทยอายุ 18–35 สองกลุ่มใช้ร่วมกัน:

1. **Casual** — เล่นระหว่างรอรถ พักกลางวัน หรือก่อนนอน ต้องการสนุกและลุ้นโดยไม่ต้องลงทุนความคิดมาก
2. **Opinionated** — ติดตามข่าว การเมือง กีฬา หรือ pop culture อย่างใกล้ชิด ต้องการ outlet แสดงความเห็นและพิสูจน์ว่าตัวเองแม่น

ทั้งคู่ใช้มือถือเป็นหลัก บริบทคือ micro-sessions 2–5 นาที ไม่ใช่ session ยาว

## Product Purpose

ภาวนาคือแพลตฟอร์มทายผลเหตุการณ์จริงด้วยคะแนน (ไม่ใช่เงิน) — community ที่รวมคนชอบฟันธงมาแข่งกันว่าใครแม่น ความสำเร็จคือ user กลับมาทายทุกวันและรู้สึกภาคภูมิใจเมื่อทายถูก ไม่ใช่รู้สึกเสี่ยงดวง

## Brand Personality

**มั่นใจ · ฉลาด · เท่**

- โทนเสียงเป็นมิตรแต่ไม่เป็นเด็ก — ไม่ตะโกน ไม่กดดัน ไม่ขายฝัน
- ใช้คำตรง ไม่อ้อมค้อม ("ทายถูก" ไม่ใช่ "ยินดีด้วย คุณได้รับ…")
- อารมณ์ขันเบาๆ ได้ในบริบทที่เหมาะ ไม่ใช่ทุกหน้าจอ
- เน้นความรู้สึก "ลุ้น" และ "รู้สึก smart" ไม่ใช่ "รวย"

## Anti-references

- **แอพพนัน/เดิมพัน** — ห้ามมีสีแดง-ทอง, countdown timer กดดัน, ตัวเลขกระพริบ, หรือ UI ที่ชวนให้รู้สึกเสี่ยงดวง
- ห้ามดู casino aesthetic แม้จะ "premium casino" ก็ตาม

## Design Principles

1. **ลุ้นได้ ไม่ต้องกังวล** — ทุก interaction ต้องรู้สึกเบาและสนุก ไม่ใช่หนักและเสี่ยง
2. **ชุมชน นำหน้าตัวเลข** — social proof (คนอื่นทายอะไร, rank ใคร) สำคัญกว่าการเน้น pool size
3. **เท่แบบสงบ ไม่ใช่เท่แบบดัง** — confidence ผ่าน whitespace และ typography ไม่ใช่ neon และ animation เยอะ
4. **Mobile-first จริงๆ** — ออกแบบสำหรับ thumb zone ก่อน ไม่ใช่ shrink จาก desktop
5. **ภาษาไทยเป็นพลเมืองชั้นหนึ่ง** — font, spacing, line-height ต้องรองรับ Thai script อย่างสวยงาม ไม่ใช่ afterthought

## Accessibility & Inclusion

- WCAG AA เป็นขั้นต่ำ — contrast ratio ≥4.5:1 สำหรับ body text
- รองรับ reduced motion สำหรับ animation ทุกตัว
- Touch target ≥44px สำหรับ interactive elements ทุกชิ้น
- ไม่พึ่ง color alone ในการสื่อความหมาย (yes/no ต้องมี label ด้วย ไม่ใช่แค่สีเขียว/แดง)
